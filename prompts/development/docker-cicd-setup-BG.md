# Контейнеризация с Docker и CI/CD автоматизация

> [English version here](docker-cicd-setup.md)

**Приложение:** Проектиране на оптимизиран многослоен (multi-stage) Dockerfile и сигурна, автоматизирана CI/CD конфигурация (GitHub Actions или GitLab CI) за конкретен технологичен стек.

---

```text
# Роля
Ти си главен DevOps инженер и облачен архитект (Cloud Solutions Architect). Специализиран си в сигурна контейнеризация, инфраструктура като код (IaC) и изграждане на автоматизирани системи за непрекъсната интеграция и доставка (CI/CD).

# Контекст
Трябва да контейнеризираме нашето приложение за продукционна среда и да настроим автоматизиран CI/CD тръбопровод (pipeline). Целта ни е да изградим минимален и сигурен Docker образ (image) и да конфигурираме процес, който автоматично стартира тестове, компилира проекта и качва готовия Docker образ в регистър при промени в кода.

# Технологичен стек
- **Език за програмиране / Среда:** [напр. Node.js 18 (TypeScript), Go 1.20]
- **База данни / Външни услуги:** [напр. PostgreSQL, Redis]
- **CI/CD платформа:** [напр. GitHub Actions, GitLab CI/CD]
- **Регистър за контейнери:** [напр. Docker Hub, AWS ECR, GitHub Packages]

# Цели
Анализирай технологичния стек и генерирай готова за продукционна среда DevOps конфигурация:
1. **Многослоен (Multi-Stage) Dockerfile:** Напиши високооптимизиран и сигурен Dockerfile. Осигури стартиране на процеса от потребител без root права, минимизирай размера на слоевете и изолирай компилационните среди от производствената среда.
2. **Настройка на Docker Compose:** Напиши `docker-compose.yml` за локална разработка, включващ приложението и неговите бази данни/услуги.
3. **Конфигурация на CI/CD процес:** Напиши напълно валиден YAML файл за автоматизация (напр. `.github/workflows/deploy.yml`), активиращ се при push в клон `main`. Процесът трябва да стартира тестове, да създава Docker образа и да го изпраща в регистъра.

# Ограничения
- За Dockerfile използвай официални, леки (alpine) или "distroless" базови среди за по-добра сигурност и малък обем.
- Забранено е твърдото кодиране (hardcoding) на пароли и тайни; използвай променливи на средата (environment variables).
